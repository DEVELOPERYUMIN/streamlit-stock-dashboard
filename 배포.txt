# Deploy
#- 개발자의 컴퓨터에서 설치한 패키지, 필요한 파일들을 실제서비스를 하고있는 컴퓨터로 옮기는 과정 

1) 필요한 페키지를 현재 상태 그대로 파일일로 저장합니다. 
pip freeze > requirements.txt 
 >> conda 쓸 경우에 안 쓰는 패키지들도 많이 깔림 : 안 쓰는 거 최대한 제거하고 올리는 게 좋음 > 가상환경 안 올리고 쓰는 패키지 'requirements.txt' 만 올리면 됨 

 2) 쓰지 않을 불필요한 파일 제거
 - gitignore 파일에 불필요하거나 공개되며 안 될 파일의 리스트를 작성함 
 - 벌써 .gitingnore 없이 git add. 을 했다면 
 - git rm -r --cached *
 3) 원격서버가 확인할 수 있는 레퍼지토리에 올림 

git init
git add . # 현재 경로에 있는 모든걸 올림 (상대적인 경로)
git commit -m "first commit" # 절취선 걷기 
git branch -M main  
git remote add origin https://github.com/DEVELOPERYUMIN/streamlit-stock-dashboard.git  # 이주소로 매핑해서 올림 
git push -u origin main
